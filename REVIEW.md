# Code Review Guidelines

クリティカルな問題のみを指摘する。軽微な指摘やスタイルの好みは投稿しない。

## 指摘すべき問題 (Critical)

- 本番環境で障害を引き起こすロジックエラー
- セキュリティ脆弱性（インジェクション、認証バイパス、シークレットのハードコード等）
- データ損失・破損につながるバグ
- git 運用パターン（squash merge、rebase 等）との整合性が取れていない処理
- 既存の動作を壊すリグレッション
- CI/CD パイプラインを破壊する設定ミス

## 指摘しない問題 (Skip)

- コードスタイル・フォーマット（Prettier / ESLint が担当）
- 命名の好み
- コメントやドキュメントの不足
- パフォーマンス改善の提案（クリティカルでない限り）
- リファクタリングの提案
- テストカバレッジの不足（テストが削除された場合を除く）

## コンテキスト

- このリポジトリは pnpm モノレポで、共有設定パッケージを提供している
- GitHub PR は squash merge で運用している
- シェルスクリプトは lefthook の git hooks として実行される
